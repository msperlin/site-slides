# Site Slides

A simple, static website framework to host HTML slides for students, organized by class and chapter.

This project uses a lightweight Flask application to dynamically discover HTML presentations, which is then frozen into a static `index.html` file to be effortlessly hosted on GitHub Pages.

## Features
- **Automatic Discovery:** Automatically scans the `slides/` directory for HTML presentations.
- **Organization:** Groups slides intuitively by Class and Chapter.
- **Auto-Generated Thumbnails:** Automatically creates image thumbnails (`.png`) for each presentation using headless Chromium.
- **Static Site Generation:** Compiles the dynamic Flask app into a single, static `index.html`.
- **CI/CD Pipeline:** GitHub Actions automatically generate thumbnails, build the site, and push the generated files back to the repository whenever changes are pushed to `main`.

## Directory Structure
To add new slides, place your `.html` presentations in the `slides/` folder following this structure:
```text
slides/
└── [Class Name]/
    └── [Chapter Name]/
        └── presentation_name.html
```
*Note: Any assets, libraries, or support files (e.g., `_files`, `libs`, `resources`) generated by presentation tools like Quarto or reveal.js are safely ignored by the scanner.*

## Local Development & Testing

1. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Run the Flask application (Dynamic mode):**
   ```bash
   python app.py
   ```
   Open `http://127.0.0.1:5000` in your browser. This dynamically renders the page based on the current contents of the `slides/` directory.

3. **Generate Thumbnails Locally:**
   ```bash
   python generate_thumbnails.py
   ```
   *Requires Google Chrome or Chromium installed on your system.*

4. **Freeze the Site:**
   ```bash
   python freeze.py
   ```
   This script creates the static `index.html` file in the root folder, which acts as the entry point for GitHub Pages.

## Deployment
Deployment is fully automated via GitHub Actions (`.github/workflows/recompile_thumbnails.yml`).

When you commit and push new slides to the `main` branch, the workflow will automatically:
1. Generate new thumbnails.
2. Freeze the `index.html` file.
3. Commit and push the newly generated files back to the repository.

You just need to point your GitHub Pages settings to serve from the `main` branch (root folder).
